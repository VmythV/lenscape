## 2. 产品概述

### 2.1 产品名称

**镜界**

### 2.2 产品一句话描述

镜界是一款面向普通用户的 AI 拍摄指导 APP，帮助用户在拍摄前获得清晰的姿势、站位、机位、角度和构图建议，让用户知道“应该怎么拍”。

### 2.3 产品定位

镜界不是相机 APP，不是修图 APP，也不是照片评分 APP。

镜界只做一件事：

> 在用户拍摄前，提供清晰、具体、可执行的拍摄指导。

用户按照指导完成拍摄，是用户自己的事情。
用户可以使用手机拍，也可以使用相机拍，也可以让别人帮拍，这些都不属于镜界的功能范围。

镜界不关心用户最终如何拍摄，也不接管拍摄过程。

---

## 3. 产品边界

### 3.1 镜界负责什么

镜界负责：

1. 帮用户选择或推荐拍摄风格。
2. 帮用户选择人物类型。
3. 帮用户选择拍摄场景。
4. 根据风格、人物类型、场景，输出拍摄指导。
5. 提供文字化拍摄步骤。
6. 提供带人物姿势边框 / 轮廓的示意图。
7. 告诉用户人物怎么站、怎么摆。
8. 告诉用户拍摄者应该站在哪里。
9. 告诉用户手机或相机应该从什么角度拍。
10. 告诉用户画面应该如何构图。

### 3.2 镜界不负责什么

镜界不负责：

1. 不负责调用相机拍照。
2. 不负责录制视频。
3. 不负责保存照片。
4. 不负责保存视频。
5. 不负责上传拍摄结果。
6. 不负责选择拍摄结果。
7. 不负责 AI 评分。
8. 不负责拍后改进建议。
9. 不负责自动修图。
10. 不负责管理用户作品。
11. 不负责用户是否重新拍摄。
12. 不关心用户使用手机、相机还是其他设备完成拍摄。

## 4. 项目背景

普通用户在拍照或拍视频前，经常遇到以下问题：

1. 不知道怎么摆姿势。
2. 不知道男生、女生、情侣、多人分别适合什么姿势。
3. 不知道拍摄者应该站在哪里。
4. 不知道手机或相机应该平视、俯拍还是仰拍。
5. 不知道人物应该放在画面哪个位置。
6. 不知道背景应该保留多少。
7. 不知道当前场景适合什么拍摄风格。
8. 不知道不同风格具体应该怎么拍。
9. 不知道景色和人物之间应该如何构图。

因此，需要一款只专注于“拍摄前指导”的 APP。

它不替用户拍摄，也不评价用户拍得好不好，而是让用户在拍摄前就知道：

1. 这个场景适合什么风格。
2. 人应该站在哪里。
3. 人应该怎么摆。
4. 拍摄者应该站在哪里。
5. 用什么角度拍。
6. 画面应该如何构图。

## 5. 产品目标

### 5.1 MVP 第一版目标

第一版聚焦：

> 拍摄前指导。

用户完成一次核心流程后，应获得：

1. 推荐或选择的拍摄风格。
2. 适合当前人物类型的姿势模板。
3. 适合当前场景的拍摄建议。
4. 文字化拍摄步骤。
5. 带人物姿势边框 / 轮廓的可视化指导图。
6. 拍摄者机位说明。
7. 手机或相机高度说明。
8. 拍摄角度说明。
9. 横屏 / 竖屏建议。
10. 人物与背景的构图建议。

### 5.2 第二阶段目标

第二阶段可以扩展到视频拍摄指导，但仍然只提供指导，不参与拍摄。

第二阶段目标是：

1. 用户选择视频拍摄场景。
2. 系统给出视频拍摄角度建议。
3. 系统给出景色构图建议。
4. 系统给出人物站位建议。
5. 系统给出横屏 / 竖屏建议。
6. 系统给出镜头机位建议。

第一版面向普通用户。

这类用户没有专业摄影经验，但希望在日常生活中拍出更好看的照片。

典型用户包括：

1. 日常拍照用户。

8. 帮别人拍照但不知道怎么指导对方的人。

### 6.2 后续扩展用户

后续可扩展到：

1. 探店博主。

3. 穿搭博主。

5. 摄影新手。

## 7. 用户痛点

### 7.1 姿势痛点

用户不知道：

1. 手应该放哪里。
2. 身体应该正对镜头还是侧身。
3. 脸应该看镜头还是看远处。
4. 眼神应该如何表现。
5. 男生适合什么动作。
6. 女生适合什么动作。
7. 情侣应该怎么互动。
8. 多人应该如何站位。
9. 同一个场景下有哪些可选姿势。

### 7.2 机位痛点

用户不知道：

1. 拍摄者应该站在正前方、斜前方、侧方还是背后。
2. 拍摄距离应该多远。
3. 设备应该放在眼平、胸口、腰部还是更低的位置。
4. 应该平视、俯拍还是仰拍。
5. 应该横拍还是竖拍。
6. 什么角度更显人好看。
7. 什么角度更能突出景色。

### 7.3 构图痛点

用户不知道：

1. 人物应该居中还是放在三分之一位置。
2. 背景应该保留多少。
3. 景色和人物之间如何平衡。
4. 当前场景中哪个方向更适合拍。
5. 如何避免背景杂乱。
6. 如何让照片更符合目标风格。

### 7.4 风格痛点

用户不知道：

1. 不同拍摄风格具体应该怎么拍。
2. 不同风格对应什么姿势。
3. 不同风格对应什么构图。
4. 当前场景适合什么风格。
5. 男生、女生、情侣、多人在同一风格下应该如何表现。

## 9. 核心用户流程

### 9.1 拍照指导主流程

```text
打开 APP
  ↓
选择拍摄指导
  ↓
选择拍摄风格
  ↓
选择人物类型
  ↓
选择当前场景
  ↓
系统推荐姿势模板
  ↓
查看拍摄前图文指导
  ↓
用户离开 APP 或参考指导自行拍摄
```

### 9.2 MVP 核心闭环

```text
风格选择 → 人物类型选择 → 场景选择 → 姿势/机位/构图指导 → 用户自行拍摄
```

### 9.3 流程终点

镜界的流程终点是：

> 用户获得拍摄指导。

不是：

1. 用户完成拍摄。
2. 用户上传照片。
3. 用户获得评分。
4. 用户保存作品。
5. 用户重新拍摄。

## 10. 功能需求

# 10.1 拍摄风格选择

## 功能说明

用户可以选择想要的拍摄风格。系统根据拍摄风格，推荐对应的姿势、机位、构图和拍摄说明。

## 第一版内置风格

第一版建议内置以下风格：

1. 韩系风格。
2. 日系清新。
3. 港风复古。
4. 街拍风格。
5. 旅行大片。
6. 情侣写真。
7. 职场形象照。
8. 证件照。
9. 探店氛围感。
10. 穿搭展示。
11. 生活随拍。
12. 高级感大片。

## 风格卡片内容

每个风格卡片包含：

1. 风格名称。
2. 风格示例图。
3. 风格说明。
4. 适合场景。
5. 适合人物类型。
6. 推荐构图方式。
7. 查看拍摄指导按钮。

## 示例

风格名称：街拍风格

风格说明：适合街道、商场外、建筑外墙、城市道路等场景，强调自然、松弛、随意和氛围感。

适合人物类型：男生、女生、不限性别、情侣。

推荐构图：竖屏，全身或半身，人物可放在画面三分之一位置。

## 验收标准

1. 用户可以看到多个内置风格。
2. 每个风格都有示例图和说明。
3. 用户选择风格后，可以进入人物类型选择或拍摄指导流程。
4. 不同风格下推荐的姿势、机位、构图必须存在明显差异。

---

# 10.2 人物类型选择

## 功能说明

用户可以选择人物类型，系统根据人物类型推荐不同的姿势模板和拍摄方式。

## 支持类型

第一版支持：

1. 男生。
2. 女生。
3. 不限性别。
4. 情侣。
5. 多人。

## 推荐逻辑

| 人物类型 | 推荐方向                 |
| ---- | -------------------- |
| 男生   | 自然、松弛、力量感、街拍感、利落感    |
| 女生   | 显身材、氛围感、柔和感、精致感      |
| 不限性别 | 走路、回头、侧身、坐姿、看远方等通用姿势 |
| 情侣   | 并肩、牵手、对视、错位站位、互动动作   |
| 多人   | 队形、层次、前后错位、左右分布、高低关系 |

## 验收标准

1. 用户可以选择人物类型。
2. 切换人物类型后，推荐内容会变化。
3. 每种人物类型至少有基础姿势模板。
4. 情侣和多人模板第一版可以做基础能力，不要求复杂动作识别。

---

# 10.3 场景选择

## 功能说明

用户选择当前拍摄场景，系统根据场景给出更合适的拍摄建议。

第一版以手动选择场景为主，不要求用户上传现场照片，也不要求调用相机识别场景。

## 场景输入方式

第一版支持：

1. 用户手动选择场景。
2. 用户输入文字描述场景，可选。

第一版不强制支持：

1. 上传图片识别场景。
2. 打开相机识别场景。
3. 实时环境识别。

## 场景类型

第一版建议支持：

1. 室内。
2. 户外。
3. 咖啡店。
4. 街道。
5. 景区。
6. 海边。
7. 商场。
8. 公园。
9. 办公室。
10. 车内。
11. 餐厅。
12. 建筑外墙。

## 场景输出

系统基于场景输出：

1. 当前场景适合哪些拍摄风格。
2. 当前场景适合横拍还是竖拍。
3. 背景是否建议保留。
4. 人物适合站在画面哪里。
5. 拍摄者适合从哪个方向拍。
6. 哪些场景元素可以作为前景或背景。

## 验收标准

1. 用户可以手动选择场景。
2. 系统可以基于场景推荐拍摄风格。
3. 系统可以基于场景调整姿势、机位和构图建议。
4. 场景选择不依赖用户上传照片。

---

# 10.4 姿势模板推荐

## 功能说明

系统根据风格、人物类型、场景，推荐拍摄姿势模板。

## 推荐输入

1. 拍摄风格。
2. 人物类型。
3. 场景类型。
4. 横屏 / 竖屏偏好，可选。
5. 全身 / 半身 / 近景偏好，可选。

## 模板内容

每个姿势模板包含：

1. 姿势名称。
2. 姿势示意图。
3. 人物姿势边框 / 轮廓图。
4. 适合风格。
5. 适合人物类型。
6. 适合场景。
7. 人物站位说明。
8. 身体动作说明。
9. 手部动作说明。
10. 面部朝向说明。
11. 眼神说明。
12. 拍摄者机位说明。
13. 设备高度说明。
14. 设备角度说明。
15. 构图说明。
16. 注意事项。

## 示例模板

模板名称：街边自然走路照

适合风格：街拍风格、生活随拍、旅行大片。
适合人物：男生、女生、不限性别。
适合场景：街道、景区、建筑外墙。

人物动作：

1. 人物从画面左侧向右侧自然走动。
2. 身体不要完全正对镜头，保持轻微侧身。
3. 一只手可以插兜，另一只手自然摆动。
4. 眼神看向远处，不必直视镜头。
5. 表情保持自然放松。

拍摄者机位：

1. 拍摄者站在人物斜前方 3 到 5 米。
2. 拍摄设备保持胸口高度或略低。
3. 使用竖屏构图。
4. 人物放在画面三分之一位置。
5. 保留部分街道和建筑背景。

注意事项：

1. 避免背景过于杂乱。
2. 不要让人物贴近画面边缘。
3. 尽量选择光线均匀的位置。

## 验收标准

1. 每个风格至少有 3 到 5 个姿势模板。
2. 每个模板必须包含示意图和文字步骤。
3. 每个模板必须包含人物姿势边框 / 轮廓图。
4. 用户可以切换不同模板。
5. 用户可以收藏模板，可选。
6. 推荐结果需要同时考虑风格、人物类型和场景。

# 10.5 拍摄前图文指导

## 功能说明

拍摄前指导是镜界的核心功能。

系统需要同时提供：

1. 文字化拍摄步骤。
2. 可视化姿势指导图。

用户在正式拍摄前，可以通过图文说明理解：

1. 人物怎么站。
2. 人物怎么摆。
3. 拍摄者站在哪里。
4. 拍摄设备放多高。
5. 用什么角度拍。
6. 人物和背景如何构图。

## 文字指导内容

文字指导包括：

1. 当前拍摄风格。
2. 当前人物类型。
3. 当前场景。
4. 人物站位。
5. 身体朝向。
6. 手部动作。
7. 面部朝向。
8. 眼神方向。
9. 拍摄者站位。
10. 设备高度。
11. 设备角度。
12. 横拍 / 竖拍建议。
13. 构图建议。
14. 注意事项。

## 图片指导内容

图片指导是在模板示意图或参考图上叠加以下元素：

1. 人物姿势边框。
2. 人物身体轮廓线。
3. 人物站位框。
4. 构图辅助框。
5. 人物推荐位置标识。
6. 背景保留区域提示。
7. 拍摄者机位标识。
8. 拍摄方向箭头。
9. 横屏 / 竖屏框线。
10. 全身 / 半身 / 近景范围框。

注意：

> 这里的图片指导是系统内置的模板图，不是用户上传的拍摄结果图。

## 可视化元素说明

| 元素     | 作用             |
| ------ | -------------- |
| 人物姿势边框 | 帮助用户理解身体外形和动作  |
| 人物站位框  | 告诉用户应该站在画面哪里   |
| 构图框    | 告诉用户人物和背景的比例   |
| 拍摄方向箭头 | 告诉拍摄者从哪个方向拍    |
| 机位图标   | 标识拍摄者位置        |
| 背景保留区域 | 告诉用户哪些背景元素应该保留 |

## 页面示例

标题：韩系咖啡店坐姿照

图片部分：

1. 展示一张咖啡店坐姿示意图。
2. 图中使用边框标出人物坐姿轮廓。
3. 图中使用框线标出人物推荐位置。
4. 图中使用箭头标出拍摄者站位和拍摄方向。
5. 图中标出桌面、窗户等建议保留的背景区域。

文字步骤：

1. 人物坐在靠窗位置，身体微微侧向拍摄方向。
2. 一只手自然放在桌面，另一只手可以拿咖啡杯。
3. 眼神看向窗外，不要直视镜头。
4. 拍摄者站在人物斜前方 45 度位置。
5. 拍摄设备略高于人物眼睛，从上往下轻微俯拍。
6. 使用竖屏构图，人物放在画面中间偏右位置。
7. 背景保留窗户、桌面和咖啡杯，减少杂乱物品。

## 验收标准

1. 每个拍摄模板都必须同时提供文字拍摄步骤和可视化指导图。
2. 可视化指导图必须包含人物姿势边框或人物轮廓。
3. 可视化指导图至少包含以下信息中的 2 类：

   * 人物姿势。
   * 人物站位。
   * 构图区域。
   * 拍摄者机位。
   * 拍摄方向。
4. 文字步骤必须具体可执行，不能只写“自然一点”“找好角度”。
5. 用户不需要理解专业摄影术语，也能根据图文完成拍摄。
6. 拍摄指导不依赖用户上传拍摄结果。

# 10.6 姿势可视化模板系统

## 功能说明

系统需要为每个拍摄模板配置一套可视化拍摄指导图，用于在拍摄前向用户展示标准姿势、人物站位、构图区域和拍摄者机位。

## 模板组成

每个可视化模板包含：

1. 原始参考图 / 示意图。
2. 人物姿势边框。
3. 人体外形轮廓。
4. 站位区域标记。
5. 构图辅助框。
6. 机位箭头。
7. 拍摄方向箭头。
8. 风格标签。
9. 配套文字说明。

## 第一版实现方式

第一版建议优先采用：

1. 人工制作的模板图。
2. AI 生成图片 + 人工审核。
3. 固定模板图 + 叠加边框、箭头、构图框。

第一版不要求：

1. 用户上传照片。
2. 识别用户真实姿势。
3. 实时人体骨架识别。
4. 实时 AR 线框叠加。
5. 拍中动态姿势纠正。

## 后续扩展

后续可以基于该模板系统扩展：

1. 更丰富的姿势模板。
2. 更多风格的可视化模板。
3. 模板图自动生成。
4. 视觉元素自动叠加。
5. 多人站位模板。

## 验收标准

1. 每个姿势模板都有对应的可视化模板图。
2. 可视化模板图中的边框、箭头、构图框清晰可见。
3. 用户可以根据图示快速理解人物怎么摆、拍摄者怎么拍。
4. 不同风格的可视化模板在姿势、构图和画面感觉上有明显差异。

# 10.7 机位与构图指导

## 功能说明

系统需要明确告诉用户：

1. 拍摄者站在哪里。
2. 设备放在哪里。
3. 用什么角度拍。
4. 人物放在画面哪里。
5. 背景保留多少。

## 机位指导维度

1. 拍摄距离：近景、中景、远景，或具体距离。
2. 拍摄方向：正面、侧面、斜前方、背后。
3. 设备高度：眼平、胸口、腰部、低角度、高角度。
4. 设备角度：平视、俯拍、仰拍。
5. 横竖屏：横屏或竖屏。
6. 人物位置：居中、左三分之一、右三分之一。
7. 景物比例：人物为主、景物为主、人物景物均衡。

## 示例

场景：景区
风格：旅行大片
人物类型：女生

建议：

1. 人物站在画面右下方三分之一位置。
2. 背景中的山、海、建筑保留在画面上方。
3. 拍摄者距离人物 4 到 6 米。
4. 使用竖屏构图。
5. 拍摄设备放在胸口高度，轻微仰拍。
6. 人物可以回头看镜头，制造旅行感。

## 验收标准

1. 系统给出的机位建议必须具体。
2. 不能只说“找好角度”，必须说明具体角度。
3. 不能只说“注意构图”，必须说明人物和背景如何摆放。
4. 用户可以根据文案和示意图直接执行。
5. 指导内容不绑定具体拍摄设备。

## 13. AI 能力需求

### 13.1 风格推荐能力

根据场景、人物类型、用户目标，推荐适合的拍摄风格。

### 13.2 姿势推荐能力

根据风格、人物类型、场景，推荐合适的姿势模板。

### 13.3 指导文案生成能力

根据姿势模板、场景和人物类型，生成清晰、具体、可执行的拍摄步骤。

### 13.4 构图建议生成能力

根据风格和场景，生成画面构图建议，包括人物位置、背景保留比例、横屏 / 竖屏建议等。

### 13.5 姿势可视化生成能力

后续可支持根据姿势模板自动生成：

1. 人物姿势边框。
2. 人物轮廓线。
3. 人物站位框。
4. 构图辅助线。
5. 拍摄者机位箭头。

第一版可先使用人工模板或 AI 生成后人工审核。

### 13.6 不需要的 AI 能力

第一版不需要：

1. 照片评分能力。
2. 拍后分析能力。
3. 成片质量判断能力。
4. 用户照片理解能力。
5. 用户照片修图能力。
6. 实时视频流分析能力。

## 14. 非功能需求

### 14.1 易用性

1. 用户不需要理解摄影术语。
2. 所有建议必须具体、可执行。
3. 页面步骤不能太复杂。
4. 拍摄指导必须图文结合。
5. 每个页面都要让用户知道下一步做什么。
6. 可视化指导图需要清晰表达姿势和构图，不应过度复杂。

### 14.2 性能

1. 首页加载时间不超过 2 秒。
2. 风格列表加载时间不超过 2 秒。
3. 拍摄方案生成时间不超过 5 秒。
4. 模板图加载需要有占位图或 loading 状态。

### 14.3 隐私

由于镜界不上传用户拍摄结果、不保存用户作品、不分析成片，隐私风险相对较低。

仍需注意：

1. 用户行为数据需要合规采集。
2. 收藏和历史记录需要用户授权或提供关闭入口。
3. 如果后续支持场景图片识别，需要单独增加隐私说明。
4. 不默认收集用户真实照片或视频。

### 14.4 兼容性

MVP 阶段建议优先支持：

1. iOS 或 Android 单平台。
2. 竖屏浏览指导。
3. 单人拍照指导场景。
4. 基础情侣 / 多人模板。

后续再扩展：

1. iOS + Android 双平台。
2. 视频拍摄指导。
3. 多人复杂姿势。
4. 平板设备适配。

## 19. 风险与应对

### 19.1 模板质量不足

风险：模板图不好看或不够实用，会影响用户体验。

应对：

1. 第一版优先精选少量高质量模板。
2. 每个风格不要追求数量过多。
3. 模板上线前人工审核。
4. 根据用户收藏、停留时长和反馈持续优化。

### 19.2 指导文案过于泛泛

风险：系统输出“自然一点”“换个角度”这类不可执行建议。

应对：

1. 限制输出格式。
2. 要求输出具体动作、位置、角度。
3. 增加模板化约束。
4. 对输出结果进行质量检查。

### 19.3 用户认为指导不够直观

风险：只有文字说明时，用户仍然不知道如何摆。

应对：

1. 每个模板必须配可视化指导图。
2. 模板图中必须有姿势边框 / 轮廓。
3. 对机位、构图、背景保留区域使用箭头和框线说明。

### 19.4 产品边界扩张

风险：后续容易扩展成相机、修图、评分、社区，导致产品范围失控。

应对：

1. 第一版严格只做拍摄指导。
2. 不接入拍摄结果上传。
3. 不做拍后评分。
4. 不做作品管理。
5. 所有新需求都先判断是否服务于“拍摄前指导”。

## 21. 第一版建议交付范围

### 21.1 建议最小可用版本

为了尽快验证产品价值，第一版建议控制在以下范围：

1. 支持 6 到 8 个拍摄风格。
2. 每个风格支持 3 到 5 个高质量姿势模板。
3. 支持男生、女生、不限性别。
4. 情侣和多人可以先作为基础模板。
5. 支持 8 到 10 个常见场景。

   * 一张风格示意图。
   * 一张带人物边框 / 轮廓的指导图。

   * 一套机位说明。
   * 一套构图说明。

8. 可选支持模板收藏。

### 21.2 第一版不建议投入

第一版不建议投入以下能力：

1. APP 内拍照。
2. APP 内录像。
3. 拍摄结果上传。
4. 拍后评分。
5. 自动修图。
6. 视频功能。
7. 实时导拍。
8. 复杂社区。
9. 复杂会员体系。
10. 专业相机参数控制。

## 23. 结论

镜界第一版应聚焦在一个清晰目标上：

> 帮助普通用户在拍摄前知道怎么摆、怎么站、怎么拍、怎么构图。

镜界的产品边界是：

> 只提供拍摄指导，不参与拍摄过程，不处理拍摄结果。

第一版核心路径是：

```text
选择风格 → 选择人物类型 → 选择场景 → 获取图文拍摄指导 → 用户自行拍摄
```

第一版的关键不是做复杂 AI，而是把“拍摄指导”做得足够清晰、具体、可执行。

其中，**带人物姿势边框 / 轮廓的可视化指导图** 是产品体验的核心差异点。它能让用户不依赖摄影知识，也能直观看懂应该如何摆姿势、如何站位、如何构图。
